# History Atlas - 交互式历史世界地图

## 项目概述
一个可交互的历史世界地图网站，用户可以通过时间轴滑块从最早有文明记录的时期（约公元前3000年）浏览到现代，在地图上实时看到各个地区的历史数据变化。

## MVP 版本要求（第一版）

### 核心功能
1. **世界地图** - 使用 Leaflet.js 或 Mapbox GL 展示可缩放、可拖拽的世界地图
2. **时间轴滑块** - 底部有一个可拖拽的时间轴，范围从约 3000 BC 到 2025 AD
3. **人口数据可视化** - 地图上用圆圈/热力图/颜色深浅展示各地区的人口数量
4. **时间轴拖动时地图实时更新** - 流畅的动画过渡

### 数据范围
- 从有大致人口记录的时期开始（约公元前3000年，美索不达米亚、埃及、中国黄河流域）
- 关键文明/地区的人口估算数据（不需要精确到国家边界，用主要城市/文明区域的点或区域即可）
- 数据来源可以用历史人口估算数据（如 Angus Maddison 的历史人口数据、UN 数据等）

### 历史人口数据节点（大致）
- 3000 BC: 美索不达米亚、埃及、印度河谷、黄河流域
- 2000 BC: 上述 + 克里特、商朝
- 1000 BC: 周朝、亚述、希腊城邦、腓尼基
- 500 BC: 波斯帝国、希腊、罗马共和国、战国时期
- 1 AD: 罗马帝国、汉朝、帕提亚、笈多、玛雅
- 500 AD: 拜占庭、萨珊、唐朝前期
- 1000 AD: 宋朝、阿拔斯、拜占庭、神圣罗马
- 1500 AD: 明朝、奥斯曼、莫卧儿、印加、阿兹特克
- 1800 AD: 清朝、英帝国、殖民时期
- 1900-2025: 现代国家数据（更精确）

### 设计要求
- **Google 风格扁平化设计** - 简洁、现代、白色背景为主
- 顶部简洁的标题栏
- 地图占据主要空间
- 底部时间轴优雅简洁
- 左侧或右侧可以有一个小面板显示当前时间点的统计信息
- 配色参考 Google Material Design

### 技术栈
- 纯前端项目（HTML + CSS + JavaScript）
- 地图库：Leaflet.js（免费开源）或 D3.js 地理投影
- 不需要后端，数据可以内嵌在 JSON 文件中
- 响应式设计但优先桌面端

### 交付物
- 一个可以直接在浏览器打开的 index.html（或简单的静态站点）
- 数据文件（JSON）
- README.md

### 注意事项
- 人口数据用历史估算即可，不需要100%精确
- 地图不需要精确的历史国界线（那是后续版本的事），先用点/圆圈/区域色块表示
- 时间轴拖动要流畅，不能卡顿
- 设计要美观现代，不要做成学术风格
